手把手教你用Hexo+Github搭建个人博客

发表于2022-03-18|更新于2022-03-20
|阅读量:

前言

哪个男孩不想拥有一个属于自己的网站呢？
拜托，有个属于自己的网站超酷的好不好。
阿强认为个人博客的本质就是分享，分享自己的经验啊，感悟啊，总结自己的错误啊，为更多的后来者分享宝贵的经验。
前进的道路不是一马平川的坦途，更多的是披荆斩棘的道路，而前辈们宝贵的经历可以使无数的后浪们不必困顿于荆棘。
其实啊，阿强认为博客和很多东西的性质是一样的，像书啊，论文啊，它们的性质都是传播知识，分享知识，只不过是换了一种形式罢了。

hexo搭建步骤

1.下载安装git
2.下载安装node.js
3.下载安装hexo
4.建创github仓库
5.设置github pages
6.部署hexo到github
7.更换主题
8.进行简单的自定义
9.发布你的第一篇文章
10.常见问题

下载安装git：

Git是一个开源的分布式版本控制系统，可以有效、高速地处理从很小到非常大的项目版本管理。
hexo是一个静态博客，每次的内容都需要生成然后更新，这样就会导致有多个版本，我们使用git来控制和上传本地网页文件到github上面去。
直接到git官网下载即可，安装选项全部默认即可。

下载安装node.js

Hexo是基于node.js编写的，所以需要安装一下node.js和里面的npm工具。

Node.js

Node.js 是一个基于 Chrome V8 引擎的 Javascript 运行环境
所以说 Node.js 不是库，是一个运行环境，或者说是一个 JS 语言解释器。
Node.js 最初的定位是提升 Ryan 自己的日常工作效率，也就是用来写服务器代码的，但是后来没有想到的是 Node.js 在前端领域却大放异彩。
说几句 Node.js 在服务器端的发展。Node.js 的诞生带给人们的是个大大的惊喜，传统上 Web 开发者，前端用 JS 写，但是写服务器端代码的时候还必须用另外一种语言，
类似 Ruby/Java/PHP 等。但是 Node.js 出现之后，JS 前后通吃了。如果去网上搜 Node.js 的资料，很多都是用 Node.js 去写服务器代码的。
Node.js 今天也成为了 Web 前端开发必不可少的基础设施。
注意，Web 前端的 JS 代码最终还是运行在浏览器中的，所以运行的时候，或者说在产品环境下，不依赖于 Node.js 。但是，Node.js 诞生以后，前端大爆发，
类似 React/Vuejs 这样的前端框架的开发环境变得非常强大和负责，Node.js 是这些开发环境运行的基础。

npm

最后来聊 npm 。Node.js 引发了前后端开发的爆发，尤其是前端。 JS 开发者众多，所以贡献开源代码的人就非常多，所有这些凝结成了 npm 这个世界上最大的软件包仓库。
npm 是 Node Package Manager 的缩写，意思是 Node 的包管理系统。Nodejs 现在如日中天，其中 npm这个功不可没。
在这里，我们要实现各种功能几乎都能找到现成的别人写好的包，直接拿了用就好了。
很多 npm 包都对应一个 Github 项目，但是如果只有代码，那么使用起来还不是特别方便。而当系统上安装好了 Node.js 之后，就会配套安装一个命令，叫做 npm 。

执行 npm install moment 就可以把 moment 这个包从 npm 的软件包仓库中下载这个包，然后安装到本地了。而 npm 的软件包仓库中，有数以万计的 moment 这样的包。
可以直接到node官网下载，直接一路next即可
npm是node.js自带的工具所以无需安装~~

公钥：公钥用来给数据加密，用公钥加密的数据只能使用私钥解密
私钥：用来解密公钥加密的数据。
公钥是可以给别人看的而私钥不可以，也就是说你把公钥给GitHub，当你上传内容时，他会通过你的私钥来验证你的身份

然后打开你的网址，看一看是不是改变啦~~

访问我们的用户名网址（也就是仓库名），诶？？？神奇的事情发生了，网站还是之前的样子呀，这是怎么回事？？
不要着急，且听我慢慢道来因为我们刚刚只是在本地进行了修改，所以我们用hexo s开启本地服务看到的是我们本地的，但是本地的话，来自五湖四海的其他人没有办法访问你的电脑上的服务，这也是为什么我们要用GitHub pages的原因，giyhub pages它可以为我们提供服务，但我们要先给它文件，也就是你的网站的源文件，我们给了它文件后当朋友们访问我们的网站时，它就会显示我们给的源文件，但是这次我们没有更新，它的手里还是我们之前给的源文件，所以没更新（没给它新的文件）就会看到我们上次给他的文件啦~~

hexo clean清除了你之前生成的东西（也就是上一次没更换主题的源文件），hexo generate的意思是生成静态页面（重新生成我们更换主题后的文件），可以用 hexo g缩写 ，hexo deploy部署文章（把新生成的文件提交给GitHub pages），可以用hexo d缩写。

进行简单的自定义：

打开网站可以看到网站标题显示hexo，我们怎么进行修改呢？
打开_config.yml文件，config意思是设置的意思，顾名思义，网站的所有设置都在这里调试。
打开文件后看到密密麻麻的参数是不是头都大了？其实大多数参数我们都用不到，不要害怕，下面由我为你一一解释。
title 网站标题
subtitle 网站副标题
description 网站描述
keywords 网站的关键词。支持多个关键词。
author 您的名字
language 网站使用的语言。对于简体中文用户来说，使用不同的主题可能需要设置成不同的值，请参考你的主题的文档自行设置，常见的有 zh-Hans和 zh-CN。
timezone 网站时区。Hexo 默认使用您电脑的时区。请参考 时区列表 进行设置，如 America/New_York, Japan, 和 UTC 。一般的，对于中国大陆地区可以使用 Asia/Shanghai。
其中，description主要用于SEO，告诉搜索引擎一个关于您站点的简单描述，通常建议在其中包含您网站的关键词。author参数用于主题显示文章的作者。
url 网址, 必须以 http:// 或 https:// 开头
permalink 文章的永久链接格式 :year/:month/:day/:title/
pretty_urls.trailing_index 是否在永久链接中保留尾部的 index.html，设置为 false 时去除 true
pretty_urls.trailing_html 是否在永久链接中保留尾部的 .html, 设置为 false 时去除 (对尾部的 index.html无效) true

修改完了网站的标题等等后，我们就获得了一个简陋的博客，浏览器打开http://localhost:4000/就可以看到啦，
还有还有！不要忘记三个命令大法哦，要记得把热乎的刚生成的源文件交给page君哦~~
