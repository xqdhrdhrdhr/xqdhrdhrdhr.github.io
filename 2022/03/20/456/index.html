前言：
哪个男孩不想拥有一个属于自己的网站呢？
拜托，有个属于自己的网站超酷的好不好。
阿强认为个人博客的本质就是分享，分享自己的经验啊，感悟啊，总结自己的错误啊，为更多的后来者分享宝贵的经验。前进的道路不是一马平川的坦途，更多的是披荆斩棘的道路，而前辈们宝贵的经历可以使无数的后浪们不必困顿于荆棘。
其实啊，阿强认为博客和很多东西的性质是一样的，像书啊，论文啊，它们的性质都是传播知识，分享知识，只不过是换了一种形式罢了。

1.下载安装git：
Git是一个开源的分布式版本控制系统，可以有效、高速地处理从很小到非常大的项目版本管理。
hexo是一个静态博客，每次的内容都需要生成然后更新，这样就会导致有多个版本，我们使用git来控制和上传本地网页文件到github上面去。
直接到git官网下载即可，安装选项全部默认即可。
2.下载安装node.js
Hexo是基于node.js编写的，所以需要安装一下node.js和里面的npm工具。
Node.js
Node.js 是一个基于 Chrome V8 引擎的 Javascript 运行环境
所以说 Node.js 不是库，是一个运行环境，或者说是一个 JS 语言解释器。
Node.js 最初的定位是提升 Ryan 自己的日常工作效率，也就是用来写服务器代码的，但是后来没有想到的是 Node.js 在前端领域却大放异彩。
说几句 Node.js 在服务器端的发展。Node.js 的诞生带给人们的是个大大的惊喜，传统上 Web 开发者，前端用 JS 写，但是写服务器端代码的时候还必须用另外一种语言，类似 Ruby/Java/PHP 等。但是 Node.js 出现之后，JS 前后通吃了。如果去网上搜 Node.js 的资料，很多都是用 Node.js 去写服务器代码的。
Node.js 今天也成为了 Web 前端开发必不可少的基础设施。注意，Web 前端的 JS 代码最终还是运行在浏览器中的，所以运行的时候，或者说在产品环境下，不依赖于 Node.js 。但是，Node.js 诞生以后，前端大爆发，类似 React/Vuejs 这样的前端框架的开发环境变得非常强大和负责，Node.js 是这些开发环境运行的基础。
npm
最后来聊 npm 。Node.js 引发了前后端开发的爆发，尤其是前端。 JS 开发者众多，所以贡献开源代码的人就非常多，所有这些凝结成了 npm 这个世界上最大的软件包仓库。
npm 是 Node Package Manager 的缩写，意思是 Node 的包管理系统。Nodejs 现在如日中天，其中 npm https://www.npmjs.com/ 这个功不可没。在这里，我们要实现各种功能几乎都能找到现成的别人写好的包，直接拿了用就好了。
很多 npm 包都对应一个 Github 项目，但是如果只有代码，那么使用起来还不是特别方便。而当系统上安装好了 Node.js 之后，就会配套安装一个命令，叫做 npm 。
npm install moment
执行 npm install moment 就可以把 moment 这个包从 npm 的软件包仓库中下载这个包，然后安装到本地了。而 npm 的软件包仓库中，有数以万计的 moment 这样的包。
可以直接到node官网下载，直接一路next即可
npm是node.js自带的工具所以无需安装~~
3.下载安装hexo：
新建一个文件夹，名字可以随意（比如我的是Myblog），打开它—对着空的文件夹右键—点击Git Bash here
然后就会弹出来一个黑框框，日后我们的绝大多数操作都要在这个黑框框了完成啦。
下面我们使用npm安装hexo，记得刚刚那个黑框框吗，输入以下指令

npm install -g hexo-cli

然后回车，静静等待安装完成~~
安装 Hexo 完成后，执行下列命令，Hexo 将会在指定文件夹中新建所需要的文件。
这里的指定文件夹也就是我们右键选择git bash here的文件夹，也就是我们新建的文件夹。
hexo init
这个命令的意思是初始化博客，完成然后，接着输入npm install安装必备的组件。
npm install
初始化完成后，指定文件夹目录下会有以下文件：
node_modules: node的依赖包
public：存放生成的页面
scaffolds：生成文章的一些模板
source：存放你的文章
themes：里面放的主题文件
_config.yml: 博客的配置文件
这样本地网站就好啦，输入hexo generate（该命令可以简写为hexo g）可以生成静态文件，然后输入hexo server（该命令可以简写为hexo s）打开本地服务，ctrl+c可以停止服务。
然后浏览器打开http://localhost:4000/，就可以看到我们的博客，万事开头难，你的第一步已经迈出来了！ ´･∀･)乂(･∀･｀以后就是像迈出第一步一样接着迈出下一步，
直至成为一个大佬~~~
4.建创github仓库：
注册github账号就不多说了（这个你们总该会吧￣△￣），不会的话自行度娘。
注册并登录之后点击New repository
注意！！！仓库名要填以下内容“你注册GitHub的用户名.github.io”不可以填别的！！比如我的名字是123456那么我的仓库名就要填以下内容“123456.github.io”
仓库要公开，要公开！！（也就是选择下面的Public选项）其他的就不用管啦
5.设置github pages：
首先，进入GitHub pages的设置页面，一开始进入如下图所示，
我们先点击change theme（改变主题），来换一个GitHub pages 的主题来测试一下

点击commit change

选择完成后就可以访问你的域名（也就是之前填写的仓库名）来看到你的页面啦~~
那么我们怎么把他原本默认的网站换成我们的hexo博客网站呢？
6.部署hexo到github：
聪明的同学可能已经发现啦，我们访问网站，访问的就是这个仓库里的文件。
那么我们如何将在本地生成的hexo文件上传到这个仓库呢？
生成ssh添加到GitHub，打开git bash然后输入下面2条指令

git config –global user.name “用户名”
git config –global user.email “邮箱”

这里的用户名输入你的GitHub用户名，邮箱输入你GitHub的邮箱。这样GitHub才能验证信息。注意：不要去掉两边的引号！！！不要去掉两边的引号！！
下面生成ssh
ssh-keygen -t rsa -C “邮箱”
这里的邮箱填写你自己的邮箱。
然后一路回车，大约要按4、5下左右。
然后它就告诉你ssh已经生成成功，根据它显示的文件夹位置找到id_rsa.pub
这个里面是公钥，相对应的id_rsa就是私钥。打开公钥文件，复制里面的内容。
或者输入cat ~/.ssh/id_rsa.pub
显示的内容就是公钥文件里的内容啦
打开github，在头像下面点击settings，再点击SSH and GPG keys，新建一个SSH，名字随便取一个都可以，把你的id_rsa.pub里面的信息复制进去。
输入ssh -T git@github.com，如果出现你的用户名，那你就成功了哦o(^▽^)o
公钥：公钥用来给数据加密，用公钥加密的数据只能使用私钥解密
私钥：用来解密公钥加密的数据。
公钥是可以给别人看的而私钥不可以，也就是说你把公钥给GitHub，当你上传内容时，他会通过你的私钥来验证你的身份
GitHub和你本地的文件就建立好联系啦，下面配置deploy，使用一键部署功能。
首先安装deploy
npm install hexo-deployer-git –save
打开博客根目录下的_config.yml文件，找到最后面。
deploy:
type: git
repo: 你的仓库地址（如下图）
branch: main
然后分别输入以下命令：
hexo clean
hexo generate
hexo deploy
hexo clean清除了你之前生成的东西，可以简写为hexo s，hexo generate的意思是生成静态页面，可以用 hexo g缩写 ，hexo deploy部署文章，可以用hexo d缩写，因为我们会经常用到这三个命令，所以我愿将其称为三命令大法(≧∇≦)。
然后打开你的网址，看一看是不是改变啦
